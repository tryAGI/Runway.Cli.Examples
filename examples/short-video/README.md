# `short-video` — one scenario → multi-shot stitched MP4

> Showcases `runway short-video`: a single scenario is expanded into a multi-shot storyboard, each shot is rendered, and the results are stitched into a final MP4 — all in one CLI call. From zero, no source images.

## 1. The prompt

What we hand to Claude — verbatim, the way a user would type it ([`prompt.md`](./prompt.md)):

> Make a short cinematic video, about 15 seconds long, of a vintage motorcycle riding through a foggy mountain pass at sunrise. Use the runway `short-video` command so the planner builds a multi-shot storyboard first and then renders each shot. Save the final stitched MP4 plus the planner's `*.plan.json`, and emit a single result.json describing the scenario, the shot list, the final video path, and the model used.

## 2. Inputs

- `RUNWAY_API_KEY` (loaded from `.env`)
- The [`runway-cli`](https://github.com/tryAGI/Runway#use-as-an-agent-skill) skill installed at `.claude/skills/runway-cli/` (done by `./scripts/setup.sh`)
- **No pre-existing assets** — `short-video` plans and renders everything from the scenario alone.

## 3. What Claude did

Guided only by the skill, Claude:

1. **Invoked `runway short-video`** with the scenario, letting the built-in planner choose shot count and composition.
2. **Captured the planner output** (`*.plan.json`) alongside the rendered MP4.
3. **Wrote `result.json`** describing the scenario, the shot list, and the final video path.

One CLI call (`runway short-video`) that fans out into one planner pass + N shot renders + a final stitch.
